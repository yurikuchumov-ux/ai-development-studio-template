# Implementation Plan — AI Development Studio v1 (Issue #1)

Статус: план, реализация не начата. Ни один пункт Scope не реализован.

Примечание: этот план сознательно не включает GitHub Actions, CI,
markdownlint или любые другие автоматические проверки — они являются
отдельной задачей после завершения Template Repository. Все критерии
завершения ниже — ручные (проверка содержимого, ссылок, независимое ревью),
без автоматизации.

Каталоги отдельно не планируются и не создаются пустыми (git не хранит
пустые каталоги) — каждый каталог появляется вместе с первым файлом
соответствующего этапа.

## Обязательные ограничения (Do not)

- `AI_OS.md` не копируется и не изменяется — он остаётся во внешнем
  репозитории `yurikuchumov-ux/ai-operating-system`.
- Правила AI Operating System не копируются и не переформулируются здесь.
  AI OS используется только как внешний Source of Truth — все документы этого
  репозитория на него ссылаются, а не дублируют.

---

## Этапы реализации

### Этап 1 — Source of Truth
Создать только:
- `standards/AI_OS_REFERENCE.md` — единственная точка, где явно указана
  ссылка на `AI_OS.md` в `yurikuchumov-ux/ai-operating-system` как источник
  истины.

Каталог `standards/` появляется вместе с этим файлом; отдельно он не
создаётся.

**Зависимости:** нет.
**Draft PR:** отдельный — `feature/issue-1-source-of-truth`.
**Параллельно с чем-либо:** нет, базовый этап.
**Критерий завершения:** ссылка на `AI_OS.md` открыта и проверена вручную
(ведёт на реальный файл в другом репозитории); в репозитории нет ни одной
копии текста правил AI OS; независимый ревьюер подтверждает соответствие
ограничению «Do not».

### Этап 2 — Документные шаблоны
Создать (каталог `templates/` появляется вместе с первым из этих файлов):
- `templates/README_TEMPLATE.md`;
- `templates/AGENTS_TEMPLATE.md`;
- `templates/PROJECT_INSTRUCTIONS_TEMPLATE.md`;
- `templates/CHANGELOG_TEMPLATE.md`.

**Зависимости:** Этап 1 (шаблоны ссылаются на `AI_OS_REFERENCE.md`).
**Draft PR:** отдельный — `feature/issue-1-doc-templates`.
**Параллельно с чем-либо:** да, с Этапом 3 и Этапом 4 — разные файлы, не
пересекаются.
**Критерий завершения:** все 4 файла существуют; каждый содержит явную ссылку
на `standards/AI_OS_REFERENCE.md`, а не копию принципов; независимый ревьюер
подтверждает отсутствие дублирования текста AI OS.

### Этап 3 — Архитектурные и плановые шаблоны
Создать (каталог `templates/adr/` появляется вместе с первым из этих
файлов):
- `templates/adr/ADR_TEMPLATE.md`;
- `templates/ROADMAP_TEMPLATE.md`;
- `templates/BACKLOG_TEMPLATE.md`.

**Зависимости:** Этап 1.
**Draft PR:** отдельный — `feature/issue-1-architecture-templates`.
**Параллельно с чем-либо:** да, с Этапом 2 и Этапом 4.
**Критерий завершения:** три файла существуют; ADR-шаблон содержит поля
Статус/Контекст/Решение/Альтернативы/Последствия; Roadmap и Backlog —
структурные шаблоны без конкретных дат/задач; независимый ревьюер
подтверждает пригодность к использованию в реальном проекте.

### Этап 4 — Шаблоны GitHub
Создать (каталог `.github/ISSUE_TEMPLATE/` появляется вместе с первым из
этих файлов):
- `.github/ISSUE_TEMPLATE/bug_report.md`, `feature_request.md`;
- `.github/PULL_REQUEST_TEMPLATE.md`.

**Зависимости:** Этап 1.
**Draft PR:** отдельный — `feature/issue-1-github-templates`.
**Параллельно с чем-либо:** да, с Этапом 2 и Этапом 3.
**Критерий завершения:** Issue-формы открываются и рендерятся в GitHub UI без
ошибок разметки; PR-шаблон содержит разделы для описания изменений, проверок
и независимого ревью; независимый ревьюер подтверждает пригодность шаблонов.

### Этап 5 — Структура Template Repository
Задокументировать (`docs/PROJECT_STRUCTURE.md`) итоговое рекомендуемое дерево
каталогов нового проекта, создаваемого из этого Template Repository, со
ссылками на конкретные шаблоны из Этапов 2, 3, 4.

**Зависимости:** Этапы 2, 3, 4.
**Draft PR:** отдельный — `feature/issue-1-project-structure`.
**Параллельно с чем-либо:** нет — интеграционный этап, требует готовности
всех трёх предыдущих.
**Критерий завершения:** каждый путь, упомянутый в документе, существует в
репозитории; независимый ревьюер подтверждает соответствие Scope Issue #1.

### Этап 6 — Инструкции для GPT, Claude Code, Codex
Три отдельных документа: `docs/agents/GPT.md`, `docs/agents/CLAUDE_CODE.md`,
`docs/agents/CODEX.md` — специфика работы каждого инструмента поверх
принципов AI OS (ссылка на `AI_OS_REFERENCE.md`, без копирования текста).
Каталог `docs/agents/` появляется вместе с первым из этих файлов.

Этот этап обязан завершиться **до** Этапа 7 (Project Cockpit), так как
Cockpit ссылается на готовые документы агентов.

**Зависимости:** Этап 1 (жёстко); Этап 5 (мягко — для согласованности ссылок
на структуру, но содержательно не блокирует).
**Draft PR:** три отдельных — `feature/issue-1-agents-gpt`,
`feature/issue-1-agents-claude-code`, `feature/issue-1-agents-codex`.
**Параллельно с чем-либо:** да — все три PR независимы друг от друга.
**Критерий завершения (по каждому PR отдельно):** документ существует, не
дублирует текст AI_OS.md; после готовности всех трёх — отдельная сверка
независимым ревьюером на отсутствие противоречий между документами.

### Этап 7 — Project Cockpit
Создать `docs/PROJECT_COCKPIT.md` — центральную точку навигации: ссылки на
шаблоны (Этапы 2–4), структуру (Этап 5), инструкции для всех трёх агентов
(Этап 6).

Этот этап начинается только после того, как структура (Этап 5), шаблоны
(Этапы 2–4) и **все три** документа агентов (Этап 6) готовы. Cockpit не
должен содержать ссылок на ещё не существующие файлы.

**Зависимости:** Этап 5, Этап 6 (все три PR).
**Draft PR:** отдельный — `feature/issue-1-project-cockpit`.
**Параллельно с чем-либо:** нет.
**Критерий завершения:** все ссылки в Cockpit проверены вручную и ведут на
уже существующие файлы (ни одной ссылки на планируемый, но не созданный
файл); независимый ревьюер подтверждает, что Cockpit — рабочая единая точка
входа в репозиторий.

### Этап 8 — Итоговая валидация против Acceptance Criteria
Контрольная точка без нового кода: построчная сверка каждого пункта
Acceptance Criteria Issue #1 (репозиторий пригоден как Template Repository,
< 5 минут на новый проект, все инструкции в репозитории, чат не источник
истины, Draft PR открыт, независимое ревью проведено, ничего не смёржено).

**Зависимости:** Этапы 1–7 полностью завершены.
**Draft PR:** нет — итоговый чек-лист или финальный Draft PR-сводка (без
merge).
**Параллельно с чем-либо:** нет.
**Критерий завершения:** ручной таймированный прогон «создать новый проект
только из материалов репозитория» укладывается в 5 минут; независимый
ревьюер подтверждает соответствие каждому пункту Acceptance Criteria.
